# Simple WebRTC Video Chat

Простое приложение для видео-звонков на двоих с использованием WebRTC, Next.js 16 и Vercel KV.

## ✨ Возможности

- 🎥 **Видео и аудио** чат между двумя пользователями
- 🆔 **Умное подключение** - введите общий ID или создайте новый
- 🔗 **Ссылки для приглашения** - или просто договоритесь об ID
- 🎯 **Клик на видео** - мгновенный полноэкранный режим
- 🔄 **Переключение камер** - фронтальная/задняя в один клик
- 🔇 **Управление микрофоном и камерой** - включить/выключить
- 📹 **Аудио-only режим** - автоподключение если камера недоступна
- 🖥️ **Fullscreen режим** с удобными круглыми кнопками
- 🔒 **P2P соединение** - прямая связь без посредников
- 🌐 **TURN fallback** - работает даже за NAT и файрволами
- ⚡ **Быстрый signaling** через Redis (Vercel KV)
- 💾 **Постоянный ID** - ваша ссылка сохраняется в браузере
- 📱 **Адаптивный дизайн** - работает на любых устройствах

## 🚀 Технологии

- **Next.js 16** с App Router и Edge Runtime
- **WebRTC** - P2P видео/аудио связь через Google STUN
- **Vercel KV (Upstash Redis)** - signaling сервер
- **TypeScript** - строгая типизация
- **Tailwind CSS 4** - современный дизайн

## 📦 Быстрый старт

### 1. Установка

\`\`\`bash
npm install
\`\`\`

### 2. Настройка Redis через Vercel Marketplace

**ВАЖНО:** Используйте только **Upstash Redis** из Marketplace!

#### Для локальной разработки:

1. Перейдите на [Vercel Dashboard](https://vercel.com/dashboard)
2. Создайте новый проект (или выберите существующий)
3. Откройте [Vercel Marketplace → Upstash Redis](https://vercel.com/integrations/upstash)
4. Нажмите **Add Integration**
5. Авторизуйтесь в Upstash (или создайте бесплатный аккаунт)
6. Создайте новую Redis базу
7. Выберите регион (рекомендуется близкий к вам)
8. Подключите к вашему проекту
9. Перейдите в **Settings → Environment Variables** проекта
10. Скопируйте три переменные:
   - \`KV_REST_API_URL\`
   - \`KV_REST_API_TOKEN\`
   - \`KV_REST_API_READ_ONLY_TOKEN\`

### 3. Создайте .env.local

Создайте файл \`.env.local\` в корне проекта:

\`\`\`bash
KV_REST_API_URL=https://your-database.upstash.io
KV_REST_API_TOKEN=your_token
KV_REST_API_READ_ONLY_TOKEN=your_readonly_token
\`\`\`

### 4. Запуск

\`\`\`bash
npm run dev
\`\`\`

Откройте [http://localhost:3000](http://localhost:3000)

## 🎮 Как использовать

### Вариант 1: По общему ID (проще)

**Оба пользователя:**
1. Откройте приложение
2. Введите одинаковый идентификатор комнаты (например, `my-room` или `team_meeting`)
   - Используйте только: a-z, 0-9, дефис (-), подчеркивание (_)
3. Нажмите **"Начать звонок"**
4. Первый создаст комнату и будет ждать, второй автоматически подключится

💡 **Совет:** Договоритесь об ID заранее - и просто введите его оба.

### Вариант 2: По ссылке (классический)

**Первый пользователь:**
1. Откройте приложение
2. **(Опционально)** Введите свой идентификатор или оставьте пустым для случайного
3. Нажмите **"Начать звонок"**
4. Разрешите доступ к камере и микрофону
5. Скопируйте ссылку кнопкой **"Скопировать ссылку"**
6. Отправьте ссылку другу

**Второй пользователь:**
1. Откройте ссылку от друга (ID подставится автоматически)
2. Нажмите **"Начать звонок"**
3. Разрешите доступ к камере/микрофону
4. Автоматически подключитесь!

💡 **Совет:** Ваш ID сохраняется в браузере и всегда остается одинаковым.

### Во время звонка:

- 🖱️ **Клик по видео** → полноэкранный режим с этим видео
- 🔄 **Клик в fullscreen** → переключение между видео собеседника и вашим
- 🔁 **Кнопка в fullscreen** → переключение фронтальной/задней камеры
- 🔇 **Микрофон** → включить/выключить звук
- 📹 **Камера** → временно выключить/включить видео
- ⛶ **Fullscreen** → вход/выход из полного экрана
- ☎️ **Завершить** → закончить звонок

## 🏗️ Архитектура

### WebRTC Flow

\`\`\`
Пользователь 1           Redis (KV)           Пользователь 2
     |                        |                        |
     |-- Создать offer ------>|                        |
     |   (+ ICE candidates)   |                        |
     |                        |<--- Получить offer ----|
     |                        |                        |
     |                        |<--- Отправить answer --|
     |<-- Получить answer ----|   (+ ICE candidates)   |
     |    (polling 5 сек)     |                        |
     |                        |                        |
     |<-------------- P2P соединение установлено ----->|
     |                        |                        |
     |-- Удалить данные ----->|                        |
\`\`\`

### API Routes (Edge Runtime)

- \`POST /api/signaling/create\` - Создание offer
- \`GET /api/signaling/get\` - Получение offer  
- \`POST /api/signaling/answer\` - Сохранение answer
- \`GET /api/signaling/get-answer\` - Получение answer + автоудаление
- \`DELETE /api/signaling/delete\` - Удаление комнаты

### Компоненты

- **VideoChat.tsx** - основная логика WebRTC
- **VideoWindow.tsx** - отображение видео (обычный/fullscreen режим)
- **WaitingRoom.tsx** - экран ожидания со ссылкой
- **CallControls.tsx** - кнопки управления (обычный режим)
- **FullscreenControls.tsx** - круглые кнопки (fullscreen)

## 🔧 Технические детали

### Оптимизации

- ⚡ **Polling 5 секунд** вместо 1 (3x меньше запросов)
- 💾 **localStorage** для постоянного room ID
- 🎯 **Edge Runtime** для минимальной задержки
- 🗑️ **Автоудаление** данных после подключения
- 🔄 **Один video элемент** для каждого потока (избегаем дублирования)
- 📝 **Фильтрация ввода** - автоматическая блокировка недопустимых символов

### STUN/TURN серверы (с TCP/TLS fallback)

Используется каскадная конфигурация для максимальной совместимости:

**Primary (UDP):**
- \`stun:stun.l.google.com:19302\` - Google STUN (быстро)
- \`stun:stun1.l.google.com:19302\` - Google STUN резервный

**Fallback (TCP/TLS через TURN):**
- \`turn:openrelay.metered.ca:80\` - TURN UDP
- \`turn:openrelay.metered.ca:443\` - TURN TCP
- \`turns:openrelay.metered.ca:443?transport=tcp\` - TURN TLS/443 (работает везде!)

💡 **Почему это важно:**
- UDP блокируется многими провайдерами и корпоративными сетями
- TCP/TLS на порту 443 проходит как обычный HTTPS трафик
- Автоматический fallback: UDP → TCP → TLS
- Бесплатный TURN сервер от [Open Relay Project](https://www.metered.ca/tools/openrelay/)

🔒 **Безопасность:**
- TLS шифрование для TURN соединений
- Public credentials (безопасны для публичного использования)

### TTL в Redis

- **Offer**: 5 минут
- **Answer**: 1 минута
- После подключения: немедленное удаление

## 🐛 Troubleshooting

### Видео не работает
- Проверьте HTTPS (или используйте localhost)
- Разрешите доступ к камере/микрофону
- Проверьте консоль браузера

### Не слышно собеседника
- Проверьте, что микрофон включен (не перечеркнут)
- Увеличьте громкость
- Проверьте настройки звука в системе

### Ошибка "Room not found"
- Комната не существует - создайте её первым
- Или комната удалилась (TTL истек или кто-то уже подключился)
- Просто нажмите "Начать звонок" еще раз

### Соединение не устанавливается

**Симптомы:** Бесконечное "Подключение..." или "ICE failed"

**Решение:**
1. ✅ **TURN fallback уже включен** - должно работать автоматически
2. Откройте консоль браузера (F12) и посмотрите ICE candidates
3. Если видите только `relay` кандидаты - используется TURN (нормально)
4. Если видите `failed` - проверьте:
   - Интернет соединение стабильно
   - Не блокирует ли антивирус WebRTC
   - Попробуйте другой браузер

**Диагностика:**
\`\`\`javascript
// В консоли браузера
pc.getStats().then(stats => {
  stats.forEach(report => {
    if (report.type === 'candidate-pair' && report.state === 'succeeded') {
      console.log('Active connection type:', report);
    }
  });
});
\`\`\`

### База данных не работает
- Проверьте переменные окружения
- Убедитесь что используете Upstash Redis из Marketplace
- Проверьте логи в Vercel Dashboard
